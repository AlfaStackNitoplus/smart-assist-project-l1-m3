Login & User Context (JSON + Data Binding)

Technical Requirements

Angular Concepts

  • Interpolation for displaying user information.
  • Two-way data binding for login inputs.

Angular Material Components

  • MatInput – for UserId / login input.
  • MatButton – for the login action.
  • MatHint – for helper or validation hints.

Functional Requirements

  • On login, the system shall identify the user using UserId from mock JSON data.
  • If a valid user is found:
  • The application shall load the user context (Name and Role).
  • The authenticated user’s details shall be available for subsequent navigation and UI rendering.
The user data model must include:
  • UserId
  • Name
  • Role
  • Error message on invalid login.

Pipes Usage (Built-in & Custom)

Technical Requirements

Built-in Pipes

  • date
  • uppercase / lowercase
  • number
  • json (optional)

Custom Pipe

  • Create a pipe to convert ticket status.
  • Pipe must be pure.

Functional Requirements

  • CreatedOn shall be displayed using date formatting.
  • Status shall be displayed using uppercase/lowercase formatting.
  • Age shall be displayed using number formatting.
  • Debug view may display JSON output (optional).
  • Ticket status shall be converted to readable labels:
  • IN_PROGRESS → “In Progress”
  • NOT_ASSIGNED → “Not Assigned”
  • FEEDBACK_RECEIVED → “Feedback Received”
